# Check Documentation

## Overview

Validate that paper documentation is complete, well-formatted, and contains required information. This ensures papers are properly documented and understandable to others.

## Parent Plan

[../plan.md](../plan.md)

## Child Plans

None (Level 4 - implementation level)

## Inputs

- Paper README.md and documentation files
- Documentation requirements
- Content quality standards

## Outputs

- Documentation validation results
- Missing sections identified
- Quality assessment
- Improvement suggestions

## Steps

1. Check README.md exists and is not empty
2. Verify required sections are present
3. Validate documentation formatting
4. Assess content completeness

## Success Criteria

- [ ] README.md presence is checked
- [ ] Required sections are verified
- [ ] Formatting issues are detected
- [ ] Content quality is assessed

## Notes

Check for required sections: Overview, Installation, Usage, Examples, References. Verify links work. Check for placeholder text. Assess clarity and completeness. Don't require perfect writing, just adequate documentation.
