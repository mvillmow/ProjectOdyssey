# Contributing

## Overview

Create a comprehensive CONTRIBUTING.md file that guides contributors through the development workflow, coding standards, and pull request process. This ensures consistent, high-quality contributions to the repository.

## Inputs

- Repository structure is complete
- Development tools are configured
- Understanding of contribution workflow

## Outputs

- CONTRIBUTING.md at repository root
- Development workflow documentation
- Coding standards and guidelines
- Pull request process and requirements
- Testing and quality guidelines

## Steps

1. Document development workflow and environment setup
2. Define coding standards and best practices
3. Explain pull request process and requirements

## Success Criteria

- [ ] CONTRIBUTING.md is comprehensive and clear
- [ ] Workflow documentation enables contributors to start
- [ ] Standards ensure code quality and consistency
- [ ] PR process is well-defined and easy to follow

## Notes

Good contributing guidelines reduce friction and improve contribution quality. Be welcoming and clear about expectations. Focus on making it easy for people to contribute successfully.
