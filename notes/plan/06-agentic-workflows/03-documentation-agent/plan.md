# Documentation Agent

## Overview
Create an AI documentation agent that generates and maintains comprehensive project documentation including API docs, READMEs, and tutorials. The agent follows Claude best practices with structured prompts and consistent documentation standards.

## Inputs
- Claude best practices for documentation generation
- Understanding of documentation standards
- Knowledge of API documentation formats
- Existing codebase to document

## Outputs
- Agent configuration with documentation standards and tools
- Prompt templates for API docs, READMEs, and tutorials
- Workflows for generating and updating documentation
- Test suite covering all documentation capabilities
- Documentation for using the documentation agent

## Steps
1. Configure agent with documentation standards and tools
2. Create prompt templates for different documentation types
3. Implement workflows for docstring generation, README updates, and tutorial creation
4. Write comprehensive tests for all documentation capabilities

## Success Criteria
- [ ] Agent configuration defines clear documentation standards
- [ ] Prompt templates cover all documentation types
- [ ] Workflows generate high-quality documentation
- [ ] API documentation is complete and accurate
- [ ] READMEs are informative and up-to-date
- [ ] Tutorials are clear and helpful
- [ ] All tests pass with good coverage

## Notes
Follow Claude best practices for consistent documentation. Use structured prompts with clear examples. Ensure documentation is accurate, complete, and maintainable. Keep tone professional but approachable.
